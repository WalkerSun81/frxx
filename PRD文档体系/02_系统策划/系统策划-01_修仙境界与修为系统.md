## 1.1 系统概述

### 系统定位

- **核心成长线**：所有玩家的必经之路，贯穿整个游戏生命周期
- **进度标杆**：境界等级是玩家实力的核心标识，决定游戏内容的解锁进度

### 与其他系统关系

```
修为获取 → 境界突破 → 解锁功能/装备位
    ↑                      ↓
  丹药加成           属性成长/技能解锁
    ↑                      ↓
  功法/宗门 ←──────── 装备强化/羁绊激活
```

### 设计目标

1. **清晰的成长路径**：玩家能够直观理解当前进度与下一目标
2. **阶段性成就感**：每个境界突破都有明确的里程碑和奖励
3. **策略深度**：突破成功率机制提供风险/收益决策点
4. **长期激励**：16个境界提供足够长的成长周期

## 1.3 修为获取机制

### 1.3.1 修为来源表

| 来源 | 公式/数值 | 占比 | 说明 |
|:---|:---|:---:|:---|
| 挂机修炼 | 修炼速度×时间 | 60% | 核心来源，支持离线收益 |
| 战斗奖励 | 100+层数×50 | 20% | 每关结算，与挂机二选一 |
| 丹药服用 | 丹方决定 | 15% | 消耗品，一次性收益 |
| 任务奖励 | 任务决定 | 5% | 每日任务、成就奖励 |

### 1.3.2 修炼速度计算公式

```
修炼速度 = 基础速度 × 灵根倍率 × 悟性倍率 × 功法倍率 × 丹药倍率 × 宗门倍率 × VIP倍率 × 羁绊倍率
```

**各乘数详细取值：**

| 乘数项 | 取值公式 | 最小值 | 最大值 | 说明 |
|:---|:---|:---:|:---:|:---|
| 基础速度 | 固定值 | 1.0 | 1.0 | 全服统一 |
| 灵根倍率 | 主灵根值/100 | 0.1 | 1.7 | 木灵根为1.7（特殊加成） |
| 悟性倍率 | min(悟性/50, 2.0) | 0.1 | 2.0 | 对数效果，悟性100时达到上限 |
| 功法倍率 | 1.0 + 功法等级×0.05 | 1.0 | 无上限 | 功法等级越高收益越大 |
| 丹药倍率 | 当前BUFF叠加 | 1.0 | 2.0 | 存在叠加上限 |
| 宗门倍率 | 1.0 + 宗门科技等级×0.1 | 1.0 | 无上限 | 依赖宗门系统 |
| VIP倍率 | 1.0 + VIP等级×0.1 | 1.0 | 2.0 | VIP10达到上限 |
| 羁绊倍率 | 道法自然=1.1，其他=1.0 | 1.0 | 1.1 | 由命格天赋决定 |

**计算示例：**

```
玩家属性：
- 主灵根：木（60值）
- 悟性：120
- 功法等级：8
- 丹药BUFF：1.2
- 宗门科技：5级
- VIP等级：6
- 羁绊：道法自然

修炼速度 = 1.0 × 0.6 × min(120/50, 2.0) × (1.0+8×0.05) × 1.2 × (1.0+5×0.1) × (1.0+6×0.1) × 1.1
         = 1.0 × 0.6 × 2.0 × 1.4 × 1.2 × 1.5 × 1.6 × 1.1
         = 3.991 ≈ 3.99

每秒修为 = 3.99（基础修为/秒）
```

### 1.3.3 离线修炼机制

| 参数 | 数值 | 说明 |
|:---|:---:|:---|

## 1.4 境界突破流程

### 1.4.1 突破前置条件

| 条件 | 要求 | 检查时机 |
|:---|:---|:---|
| 修为满 | 当前小境界修为已达上限 | 突破时 |
| 破境丹 | 背包中有对应境界破境丹×1 | 突破时 |
| 境界材料 | 背包中有境界材料×5 | 突破时 |
| 玩家状态 | 非战斗/非副本中 | 突破时 |

### 1.4.2 突破成功率计算

```
突破成功率 = 基础成功率 + 气运加成 + VIP加成 - 等级惩罚
```

**基础成功率表：**

| 突破类型 | 基础成功率 | 说明 |
|:---|:---:|:---|
| 凡人→练气 | 95% | 新手保护 |
| 练气→筑基 | 90% | 第一次大境界突破 |
| 筑基→金丹 | 80% | |
| 金丹→元婴 | 70% | |
| 元婴→化神 | 65% | |
| 化神→炼虚 | 60% | |
| 炼虚→合道 | 55% | |
| 合道→大乘 | 50% | |
| 大乘→渡劫 | 45% | 最终突破 |

**加成项计算：**

| 加成类型 | 公式 | 说明 |
|:---|:---|:---|
| 气运加成 | 气运值 × 1% | 每点气运增加1%成功率 |
| VIP加成 | VIP等级 × 2% | 每级VIP增加2%成功率 |
| 等级惩罚 | 渡劫期-10% | 仅最终阶段受惩罚 |

**成功率示例：**

```
筑基初期玩家（气运30，VIP4）突破筑基中期：
基础成功率 = 90%
气运加成 = 30 × 1% = 30%
VIP加成 = 4 × 2% = 8%
等级惩罚 = 0%

最终成功率 = 90% + 30% + 8% = 128%（上限100%）
实际成功率 = 100%
```

### 1.4.3 突破结果判定

**成功流程：**
1. 扣除破境丹×1
2. 扣除境界材料×5
3. 境界+1（小境界重置为1或进入下一境界）
4. 修为重置为0
5. 应用属性成长加成
6. 触发成功特效动画
7. 推送系统公告（可选）
8. 记录突破日志

**失败流程：**
1. 扣除破境丹×1（不返还）
2. 扣除境界材料×2.5（向下取整×5，返还50%）
3. 保留80%当前修为
4. 触发失败特效动画
5. 记录失败日志

**特殊说明：**
- 突破成功率上限为100%，超出部分不累积
- 失败后玩家可立即再次尝试突破
- 突破失败不影响当次登录的其他操作

### 1.5.2 累计属性计算

| 天赋加成 | 天赋固定值/百分比 | 不死之身=+500 |

- 天赋加成（不死之身）= +500

### 1.6.2 关键事务

**突破事务（原子操作）：**

```python
def breakthrough_transaction(role_id, target_realm_id):
    with db.transaction():
        # 1. 查询角色当前状态（加锁）
        role = Role.objects.select_for_update().get(id=role_id)
        
        # 2. 验证前置条件
        if not validate_breakthrough_conditions(role, target_realm_id):
            raise BreakthroughError("前置条件不满足")
        
        # 3. 扣减材料
        consume_items(role, [
            {"item_id": get_breakthrough_pill_id(target_realm_id), "count": 1},
            {"item_id": get_realm_material_id(target_realm_id), "count": 5}
        ])
        
        # 4. 计算成功率并判定
        success_rate = calculate_breakthrough_rate(role, target_realm_id)
        success = random.random() < success_rate
        
        # 5. 执行突破结果
        if success:
            execute_breakthrough_success(role, target_realm_id)
        else:
            execute_breakthrough_failure(role, target_realm_id)
        
        # 6. 记录日志
        log_breakthrough(role, target_realm_id, success)
        
        # 7. 推送前端
        notify_client(role)
```

**修炼事务（定期执行）：**

```python
def cultivate_transaction(role_id):
    with db.transaction():
        # 1. 查询角色修炼状态
        role = Role.objects.get(id=role_id)
        
        # 2. 计算修炼时间差
        elapsed = current_time() - role.last_cultivate_time
        
        # 3. 计算修为增量
        cultivate_speed = calculate_cultivate_speed(role)
        cultivate_amount = cultivate_speed * elapsed
        
        # 4. 更新修为
        role.current_cultivation += cultivate_amount
        role.last_cultivate_time = current_time()
        
        # 5. 检查是否满修为
        if role.current_cultivation >= role.max_cultivation:
            role.current_cultivation = role.max_cultivation
            notify_breakthrough_ready(role)
        
        # 6. 保存
        role.save()
        
        # 7. 推送修为更新
        notify_cultivation_update(role)
```

## 1.8 数值平衡性设计

### 1.8.1 成长曲线

```
整体成长曲线应符合指数增长：
- 新手期（0-4）：快速成长，建立信心
- 成长期（5-12）：稳步成长，保持动力
- 中期（13-20）：减速成长，增加深度
- 后期（21-28）：缓慢成长，凸显稀有
- 终期（29-36）：极致稀缺，荣耀象征
```

### 1.8.2 玩家留存钩子

|:---|:---|:---|

| 解锁期待 | 境界解锁新系统 | 探索动力 |
| 差距竞争 | 排行榜展示境界差距 | 社交动力 |
