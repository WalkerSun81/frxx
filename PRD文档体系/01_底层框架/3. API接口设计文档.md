## 3.1 协议规范

### 3.1.1 通信协议

本游戏采用**双通道通信架构**，根据业务场景选择最适合的协议：

| 通道类型 | 协议 | 适用场景 | 特点 |
|:---|:---|:---|:---|
| **长连接** | WebSocket over TLS | 战斗过程、挂机状态、属性变化推送、服务器主动通知 | 实时双向、低延迟、支持推送 |
| **短连接** | HTTPS (RESTful) | 登录认证、角色信息拉取、排行榜查询、商店操作、数据提交 | 无状态、可缓存、便于网关处理 |

**WebSocket 使用场景**：

- 实时战斗流程（回合制战斗状态同步）
- 挂机状态的心跳保活和进度推送
- 属性变化实时推送（如境界突破、装备强化后属性更新）
- 服务器主动推送（如系统公告、新邮件通知）
- 排行榜实时排名变化推送

**HTTPS 使用场景**：

- 玩家登录和身份认证
- 角色基础信息查询（进入游戏时全量拉取）
- 装备、背包、邮件等模块的 CRUD 操作
- 排行榜查询（可对接 CDN 缓存）
- 静态配置表拉取

### 3.1.2 请求格式

所有 API 请求均使用 JSON 格式，HTTP Header 中需包含必要的认证和协议信息。

#### HTTPS 请求格式

```json
{
  "cmd": 1001,
  "seq": 1234567890,
  "data": {
    "openid": "tap_xxxxx",
    "token": "session_token_here"
  },
  "sign": "a1b2c3d4e5f6..."
}
```

#### WebSocket 请求格式

与 HTTPS 格式一致，通过 WebSocket 帧传输。连接建立后，所有业务消息均使用该格式。

#### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| cmd | INT | 是 | 接口命令字，每个接口对应唯一的命令字编号 |
| seq | BIGINT | 是 | 序列号，客户端递增生成，用于防重放攻击和请求-响应匹配 |
| data | OBJECT | 是 | 业务请求参数，根据不同接口有不同的字段结构 |
| sign | STRING | 是 | 请求签名，算法：`MD5(token + seq + JSON(data) + secret_key)` |

**seq 防重放机制**：

- 客户端每次请求 `seq` 必须大于上一次请求的 `seq`
- 服务端记录每个玩家的最新 `seq`，丢弃 `seq` 不递增的请求
- `seq` 溢出（接近 BIGINT 上限）时需重新建立连接

### 3.1.3 响应格式

#### 成功响应

```json
{
  "code": 0,
  "msg": "success",
  "seq": 1234567890,
  "data": {
    "player_id": 12345,
    "token": "session_token",
    "role_list": [
      {
        "role_id": 1001,
        "name": "江踏雪",
        "realm_level": 3,
        "combat_power": 15800
      }
    ]
  }
}
```

#### 错误响应

```json
{
  "code": 2001,
  "msg": "灵石不足",
  "seq": 1234567890,
  "data": null
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| code | INT | 错误码，0=成功，非0=失败 |
| msg | STRING | 错误信息，成功时为 "success"，失败时为具体错误描述 |
| seq | BIGINT | 对应请求的序列号，用于客户端匹配响应 |
| data | OBJECT/NULL | 业务数据，失败时为 null |

### 3.1.4 错误码定义

#### 1xxx - 系统级错误

| 错误码 | 说明 | 处理建议 |
|:---|:---|:---|
| 0 | 成功 | - |
| 1001 | 参数错误 | 检查请求参数格式、类型、范围 |
| 1002 | 签名校验失败 | 重新计算签名或刷新 Token |
| 1003 | Token 无效或已过期 | 重新登录获取 Token |
| 1004 | 序列号校验失败（重放攻击）| 刷新连接并重新发送 |
| 1005 | 请求过于频繁（超过限频）| 降低操作频率，等待后重试 |
| 1006 | 接口不存在 | 检查 cmd 命令字是否正确 |
| 1007 | 请求方法错误 | 使用正确的 HTTP 方法 |

#### 2xxx - 资源不足错误

| 错误码 | 说明 | 处理建议 |
|:---|:---|:---|
| 2001 | 灵石不足 | 引导玩家通过任务/充值获取灵石 |
| 2002 | 铜板不足 | 引导玩家参与日常活动获取铜板 |
| 2003 | 物品不足 | 提示缺少的具体物品名称 |
| 2004 | 背包空间不足 | 引导玩家清理背包 |
| 2005 | 命格石不足 | 提示命格重置需要命格石 |
| 2006 | 强化石不足 | 提示装备强化需要强化石 |
| 2007 | 修为丹不足 | 提示使用修为丹加速修炼 |
| 2008 | 宗门贡献不足 | 引导参与宗门活动 |

#### 3xxx - 业务规则错误

| 错误码 | 说明 | 处理建议 |
|:---|:---|:---|
| 3001 | 境界未达标 | 显示当前境界和目标境界要求 |
| 3002 | 强化等级超限 | 提示装备强化已达最高等级 |
| 3003 | 装备槽位已满 | 提示卸下当前装备后再穿戴 |
| 3004 | 宝石孔未开启 | 提示需要先开启宝石孔 |
| 3005 | 宝石孔已镶嵌 | 提示先卸下原有宝石 |
| 3006 | 命格槽位未解锁 | 提示境界突破后解锁更多槽位 |
| 3007 | 命格槽位已有天赋 | 提示需要先重置原有命格 |
| 3008 | 地图未解锁 | 提示需要先通关前置地图 |
| 3009 | 层数挑战已达上限 | 提示今日挑战次数已用完 |
| 3010 | 挂机地图与角色境界不匹配 | 提示选择符合境界要求的地图 |
| 3011 | 当前状态不允许该操作 | 提示如战斗中无法进行操作 |

#### 4xxx - 频率限制错误

| 错误码 | 说明 | 处理建议 |
|:---|:---|:---|
| 4001 | 操作过于频繁 | 降低操作频率，等待数秒后重试 |
| 4002 | 挑战关卡频率超限 | 等待 1 分钟后重试 |
| 4003 | 装备强化频率超限 | 等待 30 秒后重试 |
| 4004 | 挂机操作频率超限 | 等待 10 秒后重试 |

#### 5xxx - 服务器错误

| 错误码 | 说明 | 处理建议 |
|:---|:---|:---|
| 5001 | 服务器内部错误 | 稍后重试，如持续出现请联系客服 |
| 5002 | 数据库连接失败 | 服务器维护中，请稍后重试 |
| 5003 | 缓存服务异常 | 服务器维护中，请稍后重试 |
| 5004 | 服务器正在重启 | 请稍后重试 |

## 3.2 账号模块

### 3.2.1 登录

玩家通过平台 SDK 获取 OpenID 后，调用此接口完成游戏登录。

```
接口: POST /api/login
Cmd: 1001
```

**请求参数**：

```json
{
  "openid": "tap_abc123def456",
  "platform": 1,
  "client_version": "1.0.0",
  "device_id": "xxxxxxx",
  "os_type": 1,
  "os_version": "Android 13",
  "channel_id": 1
}
```

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| openid | STRING | 是 | 平台 OpenID（TapTap/微信等）|
| platform | INT | 是 | 平台类型：1=TapTap, 2=微信, 3=游客 |
| client_version | STRING | 是 | 客户端版本号 |
| device_id | STRING | 是 | 设备唯一标识 |
| os_type | INT | 是 | 系统类型：1=Android, 2=iOS |
| os_version | STRING | 是 | 系统版本号 |
| channel_id | INT | 是 | 渠道 ID |

**响应参数**：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "player_id": 12345,
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_expire_time": 1715884800,

      {
        "role_id": 1001,

        "realm_name": "练气中期",
        "combat_power": 15800,
        "head_id": 101,
        "create_time": "2026-01-15 10:30:00"

    "announcement": {
      "title": "版本更新公告",
      "content": "...",
```

- `1001`：参数格式错误

### 3.2.2 创建角色

玩家首次进入游戏时创建角色。每个玩家最多创建 3 个角色。

```
接口: POST /api/role/create
Cmd: 1002
```

**请求参数**：

```json
{
  "name": "江踏雪",
  "head_id": 101
}
```

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| name | STRING | 是 | 角色名称，2~8 个中文字符或英文字母，不可包含特殊字符 |
| head_id | INT | 是 | 头像 ID（对应配置表）|

**响应参数**：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "role_id": 1001,
    "name": "江踏雪",
    "realm_level": 1,
    "realm_name": "凡人",

      "strength": 10,
      "vitality": 10,
      "agility": 10,
      "intelligence": 10,

      "sense": 10,
      "wisdom": 10,
      "luck": 10

      "gold": 1000,
      "copper": 5000,
      "medal_stone": 0,
      "cultivate_book": 3,
      "enhance_stone": 10

}
```

**错误响应**：

- `1001`：角色名格式错误（包含禁用词/长度不符）

## 3.6 装备模块

### 3.6.1 强化装备

对装备进行强化，提升装备的基础属性。

```
接口: POST /api/equipment/enhance
Cmd: 4001
```

**请求参数**：

```json
{
  "equipment_id": 5001,
  "use_protection": false,
  "auto_enhance": false,
  "target_level": 0
```

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| equipment_id | BIGINT | 是 | 装备 ID |

## 3.10 推送协议

服务端通过 WebSocket 主动向客户端推送事件，无需客户端轮询。

### 推送消息格式

推送消息与普通响应格式略有不同，使用 `push_cmd` 而非 `cmd`：

```json
{
  "push_cmd": 9001,
  "data": {
    "type": "attribute_change",
    "changes": {...}
  },
  "timestamp": 1715798400
}
```

### 推送命令字定义

| push_cmd | 名称 | 说明 |
|:---:|:---|:---|
| 9001 | 属性变化推送 | 角色属性、资源的实时变化 |
| 9002 | 境界突破成功推送 | 境界突破完成后的特殊通知 |
| 9003 | 等级提升推送 | 修为突破导致等级/境界变化 |
| 9004 | 物品获得推送 | 获得装备、道具时的通知 |
| 9005 | 新邮件推送 | 收到新邮件时主动通知 |
| 9006 | 公告推送 | 系统公告、紧急通知 |
| 9007 | 挂机结算推送 | 离线收益结算完成 |
| 9008 | 排行榜变化推送 | 玩家排名发生重大变化 |
| 9009 | 连接状态推送 | 异地登录、设备被踢下线 |
| 9010 | 服务器维护推送 | 维护公告、停机通知 |

---

### 3.10.1 属性变化推送 (push_cmd: 9001)

当角色属性发生变化时，服务端主动推送更新后的值。

```json
{
  "push_cmd": 9001,
  "data": {
    "type": "attribute_change",
    "timestamp": 1715798400,
    "changes": {
      "hp": {"old": 12500, "new": 12800, "delta": 300},
      "gold": {"old": 15800, "new": 15600, "delta": -200},
      "combat_power": {"old": 28500, "new": 29120, "delta": 620}
    }
  }
}
```

**前端处理**：前端接收到此推送后，替换本地缓存中的对应属性值，并触发 UI 更新（飘字动画、属性面板刷新等）。

---

### 3.10.2 境界突破成功推送 (push_cmd: 9002)

境界突破是最重要的里程碑事件，单独推送以触发特殊的表现逻辑。

```json

    "type": "realm_break_success",
    "timestamp": 1715798400,

      "name": "练气后期"
    },

      "level": 5,
      "name": "筑基初期"
    },
    "unlocked_content": {
      "map_ids": [201, 202],
      "skill_ids": [1002],
      "equipment_slots": 1,
      "talent_slots": 1,
      "feature_ids": ["realm_talent", "guild_system"]
    },
    "attribute_bonus": {
      "hp": 1500,
      "mp": 150,
      "physical_attack": 180,
      "physical_defense": 90,
      "magical_attack": 200,
      "magical_defense": 100
    },
    "play_animation": true,
```

**前端处理**：

2. 展示解锁内容弹窗
3. 更新角色面板的境界显示
4. 高亮新解锁的功能入口
