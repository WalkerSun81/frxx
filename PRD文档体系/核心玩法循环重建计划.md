# FRXX 核心玩法循环重建计划

## 1. 目标与产品定位

当前产品定位保持为“文字挂机修仙 RPG”，不改成重操作战斗游戏。

需要明确区分两个概念：

- **主要主动玩法：战斗推图。** 玩家上线后的主要操作、选择和短期目标来自推图、调整构筑和挑战 Boss。
- **主要长期资源来源：挂机修炼。** 修为主要随时间积累，但积累效率由推图进度、角色构筑和境界共同决定。

因此，“点修炼按钮”和“闯关”不再是两个并列、互相竞争的主循环。修炼负责提供时间积累，闯关负责决定积累效率、装备、突破材料和下一阶段目标。

## 2. 现状根因

现有策划同时存在以下规则：

1. 总 PRD 定义核心体验为“离线挂机收益 + 装备驱动成长 + 策略 Build 构筑”。
2. 修为系统定义挂机修炼占修为来源 60%，战斗占 20%，且两者二选一。
3. 战斗系统又把“挑战关卡 → 获得奖励 → 提升战力 → 更高层”定义为自己的核心循环。
4. 当前实现把修炼做成可反复点击/自动加速，把战斗做成一键自动结算。

这使两个系统都像主玩法，但二者之间缺少强因果关系：推图不明显影响修炼，修炼也只是直接增加数值；最终玩家不知道现在应该主动做什么。

## 3. 目标核心循环

```text
上线领取挂机收益
        ↓
修为增长 / 检查当前卡点
        ↓
挑战当前最高关卡
        ↓
获得装备、强化资源、突破材料，并提高挂机效率
        ↓
比较装备 / 调整功法与战前策略
        ↓
挑战精英或 Boss
        ↓
通关：解锁更高挂机效率与新内容
失败：根据战报选择换装、换功法、强化或短时闭关
        ↓
修为与材料满足后突破境界
        ↓
解锁新地图、技能或装备机制，回到推图
```

循环必须形成以下相互依赖：

- **闯关 → 修炼：** 最高通关层决定挂机修为效率和离线资源档位。
- **闯关 → 突破：** Boss/首通产出关键突破材料。
- **修炼 → 闯关：** 修为满并突破境界后获得属性和新构筑能力。
- **装备/功法 → 闯关：** 玩家通过选择构筑解决不同敌人，而不是只看战力。
- **失败 → 下一步行动：** 战报必须告诉玩家应该改变什么。

## 4. 系统职责重新划分

| 系统 | 唯一主要职责 | 不再承担的职责 |
| --- | --- | --- |
| 修炼 | 随时间积累修为、提供长期成长 | 不通过无限点击成为最快成长方式 |
| 战斗推图 | 主动玩法、验证构筑、提高挂机档位 | 不只是播放一次随机数结算 |
| 装备与功法 | 提供战斗策略和流派选择 | 不只是把战力数字变大 |
| Boss | 阶段考试、产出关键突破材料 | 不只是更高血量和攻击的普通怪 |
| 境界突破 | 阶段里程碑和新内容解锁 | 不作为纯随机惩罚点 |
| 离线收益 | 降低重复劳动、维持回流动力 | 不独立于推图进度无限增长 |

## 5. 数值和节奏基线

首轮实现先采用可调参数，不追求一次定死全部境界。

### 5.1 修为来源

- 挂机/离线修炼：长期修为的 60%–70%。
- 关卡首通和 Boss：20%–25%。
- 丹药及其他一次性来源：10%–15%。
- 普通关卡重复挑战不应成为无限刷修为的最优解。

### 5.2 推图对修炼的影响

- 每通过 1 个关键层，提高挂机修为效率或资源档位。
- 每 5 层给予一次可明显感知的效率提升。
- Boss 首通提供一次大幅提升，并解锁下一挂机区域。
- 收益页必须显示“当前层收益”和“下一关键层提升”，让玩家知道为什么要继续推图。

### 5.3 手动修炼按钮

推荐改为“吐纳/闭关”，而不是无限点击获得修为：

- 点击后开启一个短时修炼状态或领取已积累修为。
- 每日少量免费加速次数，作用是补齐临近突破的缺口。
- 自动修炼表示持续状态，不再是无成本的 10 倍点击器。
- 玩家想快速变强时，优先答案应是“挑战更高层提高效率”，不是连续点按钮。

### 5.4 首个体验周期

- 0–3 分钟：领取收益、穿戴首件装备、完成首次战斗。
- 3–8 分钟：第一次卡关，使用装备或战前策略解决。
- 8–15 分钟：首次精英机制和一次明显的挂机效率提升。
- 15–25 分钟：第一次 Boss，失败后调整构筑再挑战。
- 25–30 分钟：完成阶段突破，看到下一地图或新功法目标。

## 6. 按顺序执行的实施计划

### 阶段 0：统一规则和文档（P0）

工作内容：

1. 将本计划确认为核心循环的权威入口。
2. 修订总 PRD、修为系统、战斗系统和离线系统中的冲突描述。
3. 建立一张“系统产出 → 消耗 → 解锁”表，禁止同一资源出现两个互不关联的最优来源。
4. 确定第一张地图和第一个 Boss 的完整体验范围，后续地图暂不扩写。

验收：产品、界面和代码中都能用同一句话回答“玩家上线后首先应该做什么”。

### 阶段 1：先把修炼与闯关绑成闭环（P0）

工作内容：

1. 最高通关层写入挂机收益计算。
2. 普通层、关键层、Boss 层分别配置首通奖励和挂机效率提升。
3. 修炼按钮改为领取/短时闭关，不再允许无限点击成为主要提升方式。
4. 自动修炼改为状态型积累，并明确在线、离线效率。
5. Boss/首通开始产出境界突破所需材料。
6. 角色页增加“当前主目标”和唯一主按钮，例如“前往挑战第 20 层”。

验收：玩家不看说明也能理解“推得更远 → 挂机更快 → 更快突破 → 再推更远”。

### 阶段 2：让战斗产生最小有效选择（P0）

工作内容：

1. 先只做 3 个功法槽，不扩展大型技能树。
2. 第一版提供 6 个功法，覆盖爆发、生存、速度三种基础路线。
3. 增加攻、守、速三种战前策略，影响出手、伤害和承伤。
4. 灵根至少与一部分功法产生明确联动，不只提供修炼倍率。
5. 战斗结算记录每项功法贡献、伤害来源和关键回合。

验收：相近战力下，改变功法或策略能够改变 Boss 结果；测试玩家能说出自己为什么这样搭配。

### 阶段 3：重做第一张地图的战斗层次（P0）

工作内容：

1. 普通层承担基础验证和资源获取。
2. 每 5 层设置一个带单一机制的精英关卡。
3. 第一个 Boss 设置 2 个清晰机制，例如护盾阶段和狂暴阶段。
4. 在开战前展示敌方特性和可用的应对提示。
5. Boss 首次失败不直接结束体验，展示针对性调整入口。

验收：第一张地图至少出现 3 种明显不同的战斗问题，Boss 不能只靠相同构筑反复堆数值解决。

### 阶段 4：让装备掉落支持构筑（P1）

工作内容：

1. 保留当前装备框架，增加少量与三种路线对应的词条。
2. 装备详情增加新旧装备并排对比和战斗影响说明。
3. 关键层奖励改为三选一，避免所有收益都由随机掉落决定。
4. 强化失败先取消降级/归零，或加入清晰的保底进度。
5. 新装备不只显示战力升降，还提示适合哪种功法路线。

验收：玩家面对两件装备时存在真实取舍，而不是永远选择战力数字更高的一件。

### 阶段 5：统一四个现有页面的信息层级（P1）

角色页：

- 顶部显示当前阶段目标、卡点和推荐下一步。
- 修为区域解释被动收益来源及下一档提升条件。
- 突破区域显示修为、材料、Boss 条件和突破后解锁内容。

战斗页：

- 当前关卡、挂机效率提升和首通奖励放在主要位置。
- 开战前展示敌方机制、已选功法和策略。
- 失败后提供“调整功法”“更换装备”“强化一次”“短时闭关”四类行动，按实际差距推荐。

储物页：

- 增加装备对比、词条说明和构筑标签。
- 强化与卸下改为可见按钮，不依赖长按或底部小字说明。

宗门页：

- 在宗门闭环未实现前标为预告或隐藏主入口。
- 本轮不实现宗门任务、商店、排行等扩展内容。

验收：任意页面都只有一个最突出的下一步，且不会和其他页面争夺主循环入口。

### 阶段 6：补足动态反馈（P1）

工作内容：

1. 修为增长使用轻量流光和数字跳变，不让全屏持续运动。
2. 战斗加入攻击动作、受击反馈、伤害数字、暴击强调和 Boss 技能预警。
3. 装备掉落增加品质揭示和新旧装备对比过渡。
4. 突破增加蓄力、成功和失败反馈，但动画可跳过。
5. 关键层解锁时，让地图进度和挂机效率同步变化。
6. 支持减少动态效果，避免持续闪烁和过强震动。

验收：玩家能够从动画直接理解“发生了什么”和“为什么变强”，动态效果不拖慢重复操作。

### 阶段 7：测试、埋点和数值收敛（P0，贯穿执行）

最少记录：

- 修为来源占比。
- 每层挑战次数、失败次数和卡关时长。
- Boss 失败后是否更换功法、装备或策略。
- 玩家最高层、挂机时长和领取频率。
- 装备选择、强化消耗和被淘汰速度。
- 30 分钟内有效选择次数和退出节点。

首轮玩家测试：

1. 使用 5–10 名目标玩家完成 30 分钟无引导试玩。
2. 不提前解释核心循环，观察玩家第一步做什么。
3. 让玩家复述“怎样才能更快突破”和“Boss 失败后该做什么”。
4. 根据行为数据调整奖励比例、卡关间隔和反馈文案。

验收指标：

- 80% 测试玩家能复述核心循环。
- 70% 玩家进入游戏后能在 30 秒内找到当前主目标。
- 30 分钟内至少发生 3 次不同类型的有效决策。
- Boss 失败后，至少 50% 玩家会改变构筑或策略再战。
- 玩家不会把连续点击修炼按钮判断为最高效玩法。

## 7. 开发依赖顺序

```text
规则统一
  → 推图影响挂机收益
  → 修炼按钮重新定位
  → 功法/战前策略
  → 第一地图与 Boss 机制
  → 装备词条与奖励选择
  → 页面信息层级
  → 动态反馈
  → 玩家测试与调参
```

动态表现不能提前替代规则改造；如果循环关系没有建立，增加动画只会让模糊的玩法看起来更热闹。

## 8. 本轮明确不做

- 福利。
- 排行榜。
- VIP 页面。
- 修仙指南。
- 任务。
- 聊天。
- 公告和设置。
- 按境界解锁的功能宫格。
- 多宗门完整经营、PVP、复杂生活技能和大型技能树。

这些内容不影响核心循环验证，待第一张地图的 30 分钟体验通过验收后再排期。

## 9. 完成定义

本轮不是以“页面都做出来”为完成标准，而是满足以下结果：

1. 玩家知道主动玩法是推图，而修炼是由推图效率驱动的长期积累。
2. 玩家每次卡关都能找到至少两种不同的解决方法。
3. 装备、功法、Boss、境界和挂机收益形成一条互相依赖的闭环。
4. 第一张地图具备可重复验证的 30 分钟完整体验。
5. 测试数据达到阶段 7 的核心指标后，才进入后续内容扩充。
