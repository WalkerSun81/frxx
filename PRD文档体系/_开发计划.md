# 《凡人修仙录》开发计划

> **文档类型**：开发计划（Development Plan）  
> **创建日期**：2026-05-16  
> **技术栈**：Node.js + Express + SQLite + HTML5

> **平台约束**：手机端，固定 9:16 竖屏比例

## Phase 1：角色创建全流程

**目标**：新玩家能创建角色，确定灵根和命格，进入主界面。

### 任务拆解

**1.1 游客登录接口**（`POST /api/login`）
- 生成/查询 player
- 下发 JWT token
- 返回已有角色列表

**1.2 角色创建接口**（`POST /api/role/create`）
- 角色名校验（2-8字、去敏感词）
- 每个玩家最多 3 角色
- 初始化全属性为 10（凡人属性）
- 发放新手资源（灵石 1000、铜板 5000、修为丹 x3、强化石 x10）

**1.3 灵根随机接口**（`POST /api/element/random`）
- 五行各随机 1-80
- 计算主灵根、修炼倍率
- 确认前免费无限重置

**1.4 灵根确认接口**（`POST /api/element/confirm`）
- 锁定灵根，设置 `confirmed = true`
- 后续不可再改

**1.5 命格随机接口**（`POST /api/talent/random`）
- 5 个槽位按权重抽品质（凡 40%、良 30%、优 15%、极 10%、仙 4%、神 1%）
- 返回天赋列表 + 激活的羁绊预览

**1.6 命格确认接口**（`POST /api/talent/confirm`）
- 锁定命格槽位

**1.7 前端：角色创建面板**
- 角色名输入框
- 灵根雷达图（Canvas 五边形，适配 390px 宽）
- 命格星盘展示
- 确认/重随按钮

### 检查清单

- 角色创建成功并写入 DB
- 灵根随机返回五行数值
- 命格 5 槽位生成
- 前端能看到角色面板

## Phase 2：修炼 + 境界突破

**目标**：玩家能积累修为，突破境界。

### 任务拆解

**2.1 修炼接口**（`POST /api/role/cultivate`）
- 根据当前修炼速度计算每次收益
- 修炼速度公式：`基础值 x 灵根倍率 x 悟性倍率 x VIP倍率`
- 更新 `cultivate_exp` 字段

**2.2 境界突破接口**（`POST /api/role/breakthrough`）
- 检查前置条件（修为满 + 破境丹 + 突破材料）
- 计算成功率（基础 + 气运加成 + VIP 加成）
- 成功：更新境界、计算属性成长、解锁新功能
- 失败：扣除材料、保留 80% 修为
- 使用事务保证原子性

**2.3 离线收益结算**
- 登录时计算离线时长（上限 8h x 80% 效率）
- 一次性发放修为

**2.4 属性重算服务**
- 每次境界突破后重算角色全属性
- 八维基础属性 + 战斗属性（攻击、防御、气血、暴击等）

**2.5 前端：主界面**
- 境界进度条 + 名称
- 八维属性面板
- 突破按钮（材料不足时置灰）
- 成功/失败动画

### 检查清单

- 修炼 1 分钟修为有增长
- 境界突破成功/失败均按规则
- 离线登录收到收益

## Phase 3：战斗 + 地图推图

**目标**：玩家能进入地图、挑战怪物、获得掉落。

### 任务拆解

**3.1 怪物数据生成**
- 根据地图 ID + 层数计算怪物属性
- 公式已在 PRD 定义：`HP = 第一层HP x (1 + 0.04 x (N-1))`
- 普通怪 / 精英怪(x2.5) / Boss(x15) 三级

**3.2 回合制战斗引擎**（服务端运算）
- 速度排序决定先手
- 每回合：攻击方攻击 -> 伤害计算 -> 检查是否死亡
- 伤害公式：`(ATK - DEFx0.5) x 随机0.9~1.1`
- 暴击判定：`rand() < crit_rate` -> 伤害 x (1.5 + luckx0.02)
- 最多 50 回合，超时判负

**3.3 挑战关卡接口**（`POST /api/battle/challenge`）
- 校验层数解锁（只能挑战 `max_layer + 1`）
- 执行战斗 -> 返回回合日志
- 胜利：更新地图进度、发放奖励（经验+灵石+装备掉落）

**3.4 掉落系统**
- 装备掉落按怪物类型概率表（普通 70%白 25%绿 5%蓝）
- Boss 必掉对应品质装备
- 灵石按层数递增

**3.5 前端：战斗界面**
- 回合制动画（攻击 -> 飘字 -> 敌方反击）
- 血条动态变化
- 暴击特殊颜色提示
- 战斗结果弹窗

**3.6 前端：地图选择界面**
- 7 张地图的解锁状态
- 通关层数显示
- Boss 层高亮

### 检查清单

- 能打 1 层小怪并获胜
- 击败 Boss 有装备掉落
- 网络断开会停止战斗

## Phase 4：装备系统

**目标**：装备能掉落、穿戴、强化，战力实时更新。

### 任务拆解

**4.1 装备生成服务**
- 根据品质生成基础属性 + 随机额外词条
- 宝石孔数量随机

**4.2 装备穿戴/卸下接口**
- `POST /api/equipment/equip` — 槽位对应，同槽自动替换
- `POST /api/equipment/unequip`

**4.3 装备强化接口**（`POST /api/equipment/enhance`）
- 消耗强化石 + 铜板
- 成功率表：+1~+3=100%，+4~+5=95%... +19~+20=15%
- 失败处理：降级 / 保护符防止降级
- 强化后属性按百分比增长

**4.4 背包系统**
- `t_bag` 表：道具管理
- 背包查询、道具叠加、同模板自动合并

**4.5 战力计算服务**
- 境界(40%) + 装备(35%) + 灵根(10%) + 命格(10%) + 技能(5%)
- 装备变化时自动重算并推送

**4.6 前端：装备面板**
- 11 个槽位的装备栏
- 点击槽位 -> 选择背包装备 -> 穿戴
- 强化按钮 + 成功率 + 保护符勾选

### 检查清单

- 装备能穿戴且属性变化
- +5 强化消耗资源正确
- 战力公式结果合理

## Phase 5：宝石 + 生活技能

**目标**：宝石可镶嵌合成，生活技能可采集制作。

### 任务拆解

**5.1 宝石接口**
- `POST /api/gem/mount` — 镶嵌
- `POST /api/gem/unmount` — 卸下
- `POST /api/gem/compose` — 2 颗同级合成 1 颗高级

**5.2 生活技能采集**（`POST /api/life/collect`）
- 每日次数限制（Lv.1=50, Lv.5=100）
- 概率产出材料（按专精等级不同概率表）
- 经验增长

**5.3 生活技能制作**（`POST /api/life/craft`）
- 配方校验、材料扣除
- 品质随机（公式已有）
- 产出装备/丹药

**5.4 配方数据库**
- 5 类制作 x 每类约 6-8 个配方
- 以 JSON 配置文件存储，不硬编码

### 检查清单

- 2 颗 1 级宝石合成能出 2 级
- 采集每日次数正常
- 炼丹产出丹药能使用
